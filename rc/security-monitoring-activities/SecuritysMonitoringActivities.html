Known-bad Internet Protocol (IP)

The known-bad Internet Protocols or IP addresses are sources of various threats. In most cases, these IP addresses are known to perform various malicious activities, such as:

  • Active source of malware
  • Active role in botnets
  • Active role in sending phishing Emails
  • Known to be Command-and-Control (CNC) systems managing one or more botnets
  • Any type of malicious cyber activity

Known-bad IP addresses are also known as Domain Name System Blacklists. In some cases, there can be a possibility in which a legitimate IP address is blacklisted. One of the reasons can be that this IP address is sending out spam, which it is not. There can be a possibility that too many Emails sent out, and therefore, the IP address got blacklisted.

Malicious Payload

A malicious payload is a piece of code that can be dormant for several days or even months before it is triggered. It is usually embedded within another application or piece of code, which, when triggered, activates a malicious payload. The intent of this could vary, depending on its nature. For example, some can be designed to:

  • Steal data
  • Monitor activity
  • Delete files
  • Download more malicious files
  • Run a background process

There are several methods to deliver a malicious payload to a target system. Email is one of the key methods or delivery vector for this. There is usually an attachment or a link that is delivered via an Email, which, when either clicked or opened, will trigger the malicious payload.

Emails with malicious payloads come from unknown sources. However, it could also be possible that an Email came from a known source but contains an unknown attachment. There is a possibility that the Email was spoofed, which is a common practice followed by attackers. They can compromise a messaging server or network of an organization and then use the messaging server to route their emails with malicious payloads. For the recipient, the Email seems to come from a known source and, therefore, falls into the trap of opening the attachment.

Domain Key Identified Mail (DKIM)

As mentioned above, spoofing is one of the most common problems with Email. For example, John sends an Email to Mary, but the contents of the Email states that it has come from James.

Domain Key Identified Mail (DKIM) helps you to prevent spoofed Email leaving your messaging servers. When an Email is outbound of the messaging server, it adds an encrypted signature to the header of the Email. When this Email is received by a messaging server, it needs to verify the message for its authenticity. The message can only be verified after the recipient messaging server decrypts the message.

Sender Policy Framework

A Domain Name System (DNS) is a network service that translates a hostname to its numeric Internet Protocol (IP) address and vice-versa. Although largely associated with the Internet to find hosts in the massive public network, private organizations use DNS as well so that hosts and their respective services can be located in a corporate network.

DNS relies on zones that contain resource records (RRs), which are used to ensure a successful resolution of hostnames to its IP address. For DNS servers facing the Internet, its RRs contain hostnames that are resolvable in the public network. These hostnames are typically web servers, mail relay servers, and FTP servers.

An important DNS record is the Sender Policy Framework (SPF). This record identifies specific mail servers that are allowed to send e-mail messages from that organization’s domain. The SPF is an important record against senders of junk mail who send e-mail messages with a spoofed “From address” because the SPF record lists which IP addresses or addresses are authorized to send Emails from that domain.

Domain-based Message Authentication, Reporting, and Conformance (DMARC)

You may have, in the past, received an Email that has come from a known address. However, the sender of the message, if you check in person, denies sending such an Email to you. This means that someone has forged the From address of the Email, which indicates that the spammers are using your domain name to send spam messages.

Domain-based Message Authentication, Reporting, and Conformance (DMARC) is a protocol that uses SPF and DKIM. It is used to determine the authenticity of the Email. If DMARC is enabled on a domain, its messaging server will verify the incoming Emails to ensure that they are authentic. If the DMARC policy is implemented in the domain, and an Email does not pass its verification process, then the DMARC policy is triggered.

Two possible actions can take place:

  • An Email passes the verification process: This indicates that the Email was authenticated by SPF or DKIM (could be both), and the domain in the From field is authenticated by DMARC.
  • An Email does not pass the verification process: There can be three different types of actions that can be defined when an Email does not pass the verification process:

- No action is taken

- Email is marked as SPAM

- Email is rejected by the server

The DMARC policy can be implemented via the DNS by any organization.

Phishing

Phishing is an act of obtaining confidential information from the user by using a fake identity. The attacker poses as a legitimate and trustworthy entity who is seeking information for genuine reasons; however, their identity is never disclosed. If the user provides the required information (thinking that the attacker is as an authorized entity), the attacker then will use this information for harmful activities. Generally, phishing takes place using Email. An attacker would pose as an authorized entity, and demand information in reply to that mail.

Phishing can be conducted through various methods:

  • In-person
  • Through a malicious Website
  • Through E-mail malware attachments

Reasons for Successful Phishing Attacks

Lack of foresight and ignorance are probably the two big reasons why users fall into the phishing trap. Several users, typically those who are not tech-savvy, get lured by an E-mail that asks them to log on to a seemingly legitimate website to claim a large amount of money as a prize. They are requested to provide their username and password on said website, which is a spoofed one made to resemble the original.

Phishing and its types

Watering Hole

A Watering Hole is a more complex type of phishing attack. In this type of attack, an attacker infects a website that is often visited by the target users. The website is infected with malware that is injected into the user’s system when they connect to the website. It is a multi-phased attack in which the attacker first profiles the target, who are employees of an organization or a government agency. In the profiling stage, the attacker learns about the website these employees frequently visit. Then, the website is infected with malware. When the users visit the website, their systems are infected.

The majority of the phishing attacks are performed using E-mails, and mostly in the form of spam. Phishers create a database of millions of E-mail accounts, usually from social networking Websites, and then send E-mails to these accounts. Over the years, spam E-mails have been well-written and well-designed, making them more convincing. Of late, along with messaging, spam SMS has picked up its pace on mobile phones.

Whaling

Similar to phishing and spear phishing, but the size of the target is much bigger. Whale phishing is meant to target high-profile candidates, such as CEO or CIO in an organization, or maybe a well-known and established person, such as a film star. High-profile people have high-profile secrets to keep - personal or business-related, which can be used against them. Remember, phishing is just another form of social engineering, and your convincing power plays a major role. The attacker may create a sense of urgency in an E-mail and force a person to click a URL embedded in the E-mail.

Let’s take an example - using whale phishing, you could get your hands on an executive’s official E-mail, and official bank account credentials. You cannot only approve monetary transactions but also perform them.

To prevent whale phishing, you need to ensure that you build enough technical and detection controls. Not only in the office, but they also need to be secure at home as well as when they are using mobile phones.

Spear Phishing

Spear phishing, unlike general phishing, targets specific individuals and companies. This entity could be an individual or several individuals. An attacker may target a top executive of an organization to steal information, and the spear-phishing may be used to install malware on to his or her system, and eventually get into the network. The spear-phishing Emails are usually designed to look like they originate from a well-known company or a Website. For example, as a top executive of a firm, you may receive an Email from eBay asking you to reset your account password. The E-mail includes a URL to click and states that there have been unauthorized login attempts, and thus, you should reset your password. Otherwise, your account will be locked. In this example, the E-mail was designed to create a sense of urgency.

Digital Signatures

Digital signatures are used for signing emails to effectively place a unique signature on an Email to help with verification of the sender. The act of signing is effectively a digital signature. The idea is to have the message readable to everyone. Still, the clever part is checking whether the message has been changed by someone other than the author of the Email, thereby helping to thwart man-in-the-middle attacks and maintain the message integrity.

Two keys are used in a digital signature - a private key and a public key. A private key is used when a message is signed. A public key is available with the recipient of the message and is used to verify message authenticity.

To be able to create a digital signature, you need to use your private key. For example, John, after receiving the message with the digital signature, needs to use the sender's public key to decrypt the message. You cannot create a digital signature with your own or recipient’s public key. Only John can use his private key to create a digital signature. You cannot use someone else’s private key.

Email Signature Block

A signature block is always towards the end of an Email, and it signifies the names and details of the sender. However, it is important to understand that the Email signature block can easily be forged or copied. It is not secure at all, and therefore, digital signatures can be used as a replacement for the more typical Email signature block. This is because digital signatures cannot be tampered with.

Embedded Links

Emails contain embedded links, which can either be valid or malicious. Most spam or spoofed Emails contain embedded links, which are malicious. When the recipient clicks on the embedded link, different types of actions can take place, depending on how the link has been programmed.

  • The user can be navigated to a Website, which infects your system with malware or ransomware. It could also be that a keylogger is downloaded on the recipient’s system.
  • Alternatively, malware is directly downloaded onto the recipient’s system.

As a user, you should have reason to suspect whether an embedded link is malicious or not. If the Email is from a known person, there is no reason for you to suspect. However, you can still hover over the embedded link and verify the URL, whether it is the same as the embedded link. The rule of clicking the embedded link is that click only if you are expecting it.

Impersonation

Impersonation is the act of pretending to be someone else. In a cybersecurity context, impersonation is using a fake identity. The impersonator is an attacker, who shows a fake identity to the victim. If the attacker is wrongly identified as a trustworthy entity, then the victim is likely to disclose confidential information. This information could be passwords, information related to financial transactions, etc. This information can be used by the impersonator to perform a harmful activity.

Headers

Each Email has a header that contains loads of information. It contains not only the To and From Email address, but also the information regarding the full hop trail that the message has taken to reach the recipient’s mailbox.

This kind of information can be good at tracking down an Email source in the case of a security incident. The Email header will always contain the true source IP address from where the Email originated.

In Microsoft Outlook, you can open an Email, click File, and then click Properties. The Internet headers section contains the details of the Email To and From along with the complete hop trail on the Properties dialogue box.

Exercise 3 - Network

Malware is a piece of code that has been programmed into a form of executable or code that has been embedded within legitimate software. The code or application is programmed to do something. When assembled, it is converted into binary format.

When you reverse engineer it, it is converted to code mnemonics, which can be further evaluated to understand how the software is programmed to work.

In this exercise, we will discuss different aspects of network monitoring activities.

Task 1 - Malware Reverse Engineering

There are different methods to reverse engineer software or malware. They fall into different categories, which are:

  • Disassemblers: It can reverse engineer the software into assembly code. It can also convert the binary code into native code. One example is IDA Pro.
  • Debuggers: It can manipulate the software execution to understand what it is designed to do. Examples are x64dbg, Windbg, and GDB.

Memory

As a usual practice, attackers generally will leave no trace of their existence. However, forensic investigators will attempt to investigate the memory dump, which can contain much information, such as:

  • Running processes
  • Running executables
  • Open ports
  • Communicating IP addresses
  • Open files
  • Running system activities

When software or malware is executed, it is first loaded into memory, which helps forensic investigators identify attacks that have been conducted.

Physical memory also contains a lot of information, such as:

  • Decrypted software and applications: If malware is hidden in legitimate software, then it needs to be decrypted first.
  • User credentials: Any user credentials used can also be stored in physical memory.
  • Entered content: Any information copied into the clipboard or entered into a form field will be stored in physical memory.

All this information from memory can be utilized to trace an attack. However, memory forensics is not something that a beginner or novice should handle. Rather, it needs to be handled by properly trained forensics investigators who are specialized in memory forensics.

System and Application Behavior

Each system and application works with a certain type of behavior. This behavior can either be good, which means that as per the rules and defined policies, i.e., it can function as intended. Contrary to this, behavior can also be anomalous, which can be driven by malware or certain compromised services.

Known-good Behavior

Several tools are now focusing on known-good behaviour, rather than unknown-behavior. Rather than focusing on unknown-behavior, which can be malicious, it is easy to focus on the known-good behavior. For example, in a firewall, you can choose to allow only known-good applications and services.

Anomalous Behavior

In simple plain English, anomalous behavior is a type of behavior that deviates from the defined guidelines and policies. The anomalous behavior can be determined by using certain specialized tools, such as User and Entity Behavior Analytics (UEBA) tools.

If there is a deviation from the baseline set by the UEBA tool, it can be considered anomalous. Some examples are:

  • Attempting to access folders on which the user does not have permissions
  • Unusual login timings and locations
  • A single user login logged in several systems
  • Sending a large amount of data to a remote system

Exploit Techniques

Vulnerabilities can exist in an operating system, application, or firmware. Depending on where they exist, exploits can be designed accordingly. Exploits are never generic and are designed to target a specific vulnerability. There are applications available that serve as a platform to run exploits. Metasploit includes several exploits that are designed to exploit a specific vulnerability.

File System

After you retrieve and secure evidence, you’re ready to copy evidence media and analyze data. The first step when performing forensics is to keep and preserve evidence, and then conduct analysis only on a copy of the data.

Files that are deleted are still kept on disk until a new file is saved to the same physical location which would overwrite em. As such, if needed those files can still be retrieved.

The next step is analyzing data and searching for information related to the complaint. Data analysis can be the most time-consuming task, even when you know exactly what to look for. The method for locating evidentiary artifacts is to search for specific known data values. Data values can be unique words or nonprintable characters, such as hexadecimal codes.

There are also printable character codes that can’t be generated from a keyboard, such as the copyright (©) or registered trademark (®) symbols. Many digital forensics programs can search for character strings (letters and numbers) and hexadecimal values, such as A9 for the copyright symbol or AE for the registered trademark symbol. All these searchable data values are referred to as “keywords.” With Autopsy, you can search for keywords of interest in the case.

User and Entity Behavior Analytics (UEBA)

User and Entity Behavior Analytics (UEBA) uses machine learning and deep learning to track abnormal and malicious user behavior. It also evaluates the behavior of the network devices and systems on a network.

When you use a traditional security tool, it will work based on one of the following:

  • Rules or pre-defined conditions
  • Attack patterns
  • Signatures or updates

UBEA, on the other hand, does not work with the traditional methods. It uses baselining as the key method. In baselining, a model behavior is determined in the network environment. You can use UBEA to determine the model behavior of various systems and network devices, such as servers and routers.

Once model behavior is established, any deviation from it can be determined as anomalous and a security threat.
