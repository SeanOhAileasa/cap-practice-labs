Exercise 1 - Permissions

Operating systems work with permissions on files and directories. However, the issue is that permissions apply the same for all the users in a system. For example, if you assign the Read permission, then it would apply to all users. Each user can be assigned different permissions using Access Control Lists (ACLs). Setting up access control lists on files and directories allows you to control access to files and directories strictly.

In this exercise, you will learn to assign permissions using ACLs.

Learning Outcomes

After completing this exercise, you will be able to:

  • Implement the Access Control List 

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • PLABKALI01 - (Kali Linux 2019 - Linux Kali)

Task 1 - Implement the Access Control List

ACLs can be configured in four different ways, which are as follows:

  • Per-user basis
  • Per group basis
  • Using an effective right mask
  • For users other than the ones in the group for a file

In this task, you will learn to implement the access control list. To do this, perform the following steps:

Step 1

Ensure that all the required devices are powered on. Connect to PLABKALI01.

On the desktop, click the Terminal application from the application bar.

Step 2

You need to first to check if the kernel supports ACL. Type the following command:

cat /boot/config-5.2.0-kali2-amd64 | grep _ACL

Press Enter.

Step 3

The output of the command is displayed.

Step 4

Clear the screen by entering the following command:

clear

You can search for packages on Kali Linux using the apt-cache command, which searches for package names as well as their description. Type the following command:

apt-cache search acl | grep acl

Press Enter.

Note: Without the grep acl command, you may get too many results. Therefore, you filter the results, and you can use grep acl after piping.

Step 5

The output of the apt-cache command is displayed. Notice that the acl package is listed in the results.

Step 6

Clear the screen by entering the following command:

clear

Let’s now apply the ACLs. To do this, you need to first create a file in the / directory. Type the following command:

touch plab.txt

Press Enter.

Step 7

The file is now created. Notice that no output of the touch command is returned.

Step 8

You need to verify the permissions on the plab.txt file. Type the following command:

ls -l

Press Enter.

Step 9

The permissions for plab.txt are now displayed. Notice that other users will be able to read the file.

Step 10

Clear the screen by entering the following command:

clear

You need to ensure that other than the root user, no one has read, write, and execute permissions for the plab.txt file. Type the following command:

chmod 700 plab.txt

Press Enter.

Step 11

Notice that the chmod command does not return an output.

Step 12

You need to verify the permissions on this file again. Type the following command:

ls -l plab.txt

Press Enter. The permissions for plab.txt are now displayed. Notice that other users do not have permissions on this file. The Read permission that was earlier set on the file has now been removed.

Step 13

Clear the screen by entering the following command:

clear

You can verify the permissions on this file using the getfacl command. Type the following:

getfacl plab.txt

Press Enter.

Step 14

Notice that the root user is the only one that has the read, write, and execute permissions. Neither the group nor the user has these.

Step 15

Let’s create a user in Kali Linux. To create a user, type the following command:

adduser --home /mary mary

Press Enter.

Note: The --home parameter will add the home directory for the user that you are creating.

Step 16

You will be prompted to provide the password. In the New password prompt, type the following:

Passw0rd

Press Enter.

In the Retype new password prompt, type the following:

Passw0rd

Press Enter.

Step 17

When requested to provide information, press Enter five times and then type Y to validate it.

Step 18

Clear the screen by entering the following command:

clear

A user named mary is now created. Let’s assign permissions to mary on the plab.txt file.

setfacl -m user:mary:rwx plab.txt

Press Enter.

Step 19

Notice that no output is returned, which means that the command was executed successfully.

Step 20

You can now verify the permissions on the plab.txt file. Type the following command:

getfacl plab.txt

Press Enter.

Step 21

Notice that the user mary now has read, write, and execute permissions.

Step 22

Clear the screen by entering the following command:

clear

You can also verify the permissions with the ls -l command. Type the following command:

ls -l plab.txt

Press Enter.

Step 23

Notice that permissions are also assigned to the user, which in this case is mary.

Exercise 2 - Software Restriction Policies

Software restriction policies are a policy-driven mechanism that provide administrators the ability to identify and control applications that can run on a Windows computer. The policies protect Windows clients starting from Windows XP and later versions from security threats such as viruses and Trojan host programs.

The SRPs can either be applied to a local Windows system using Local Security Policy or through the domain using Group Policy. With the SRPs, an administrator can blacklist or whitelist applications that the users can run. If an application is whitelisted, the user will be able to run the application. However, if the application or a specific path that contains the executables is blacklisted, then the application executable or all executables within the defined path will be blacklisted.

With SRPs, an administrator can enhance the security posture of a domain by allowing or disallowing only certain applications.

In this lab, you will learn how to create a software restriction policy on a local computer-based on the file path and hash rules.

Learning Outcomes

After completing this exercise, you will be able to:

Exercise 4 - Sandboxing

By default, in a Windows environment, software is installed on the hard drive directly. With a sandbox solution, software is instead installed and run from a sandbox environment, which means that they are not written to the hard drive. Windows 10 provides an inbuilt sandbox solution that allows you to run software in a controlled environment. Windows Sandbox is a virtual machine with a small footprint. You can test software within the sandbox, and after you close it, the contents are deleted automatically.

In this exercise, you will learn to work with Windows Sandbox.

Learning Outcomes

After completing this exercise, you will be able to:

  • Work with Windows Sandbox

Your Devices

Task 1 - Windows Sandbox

Windows Sandbox is available in Windows 10 Pro and Enterprise. It has certain requirements that you need to meet to be able to use:

  • Operating System: Windows 10 Pro or Enterprise
  • Virtualization: Hardware
