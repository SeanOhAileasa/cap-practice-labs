Exercise 1 - Enumeration

Enumeration allows you to establish an active connection with a target. Your intent is to extract different types of information from the target. Some of the information that you can extract is:

  • Usernames
  • Group Names
  • Hostnames
  • Network Shares and Services
  • Routing Tables
  • Web Application
  • Web Servers
  • SNMP Information
  • DNS Information

Various types of enumerations can be performed. Some of these are:

  • Windows Enumeration
  • Linux Enumeration
  • NetBIOS Enumeration
  • SNMP Enumeration
  • LDAP Enumeration
  • NTP Enumeration
  • SMTP Enumeration
  • DNS Enumeration

In this exercise, you will learn to use some of the enumeration techniques using Windows-based tools.

Learning Outcomes

After completing this exercise, you will be able to:

  • Use the Hping Command
  • Understand Active vs. Passive Enumeration

Task 1 - Nmap

Network Mapper, known as Nmap, is a network and host discovery tool. It is one of the most widely used tools for various activities, such as:

  • Discovering hosts, services, and ports
  • Fingerprinting operating systems
  • Enumeration
  • Discovering vulnerabilities on the local and remote host
  • Finding the IP address of a remote system

Using Nmap, you can scan for targets by:

  • Scanning for a single IP: nmap 192.168.0.1
  • Scanning for a host by using its name: nmap host1.plab.com
  • Scanning an entire subnet: nmap plab.com/24, nmap 192.168.0.0/24, nmap 192.168.0.*
  • Scanning for a range of IP addresses: nmap 192.168.0.1-10
  • Scanning for a range and a system outside the range: nmap 192.168.0.1, 1.10

In this task, you will use Nmap to identify the live systems on a network.

Using the port scan method, you can determine the TCP or UDP port that is being used. You can either scan for the entire range of ports, which are from 1 to 65535, or can scan for specific ports. When scanning for ports, you need to be aware that there can be different states of a port:

Open: An application is listening for connections on this port.

Closed: The messages were received, but no application is listening on the port.

Filtered: The messages were not received, and the state of the port could not be determined. This state occurs when some type of filtering is being used on the port.

Unfiltered: The messages are received, but the state of the port could not be determined.

Open/Filtered: The port was either filtered or open, but Nmap was not able to determine the state.

Closed/Filtered: The port was either filtered or closed, but Nmap was not able to determine the state.

Step 5

The output of the hping3 command will be limited to five times.

Step 6

Clear the screen by entering the following command:

clear

You can also use the hping3 command to scan for a specific TCP port. You need to specify the port number with the - - scan parameter. Type the following command:

 hping3 intranet --scan 80 -S

Press Enter.

Note: The -S parameter sets the SYN flag.

Step 7

Notice that in the flags column, S and A are mentioned. This means that the SYN+ACK response was received from the target system.

SYN+ACK is part of the TCP 3-way handshake process between two systems, which has the following steps:

  • Host1 sends a TCP SYN (synchronize) packet to Host2
  • Host2 receives Host1's SYN (synchronize) packet
  • Host2 sends an SYN (synchronize) -ACK (Acknowledgement).
  • Host1 receives Host2's SYN+ACK
  • Host1 sends ACK (Acknowledgement)
  • Host2 receives ACK (Acknowledgement).

Step 8

Clear the screen by entering the following command:

clear

You can also scan for multiple ports using the hping3 command. To do this, type the following command:

hping3 intranet --scan 22,80,443 -S

Press Enter.

Step 9

Notice the output. Only port 80 has responded. The output only displays the ports if the SYN+ACK response is received.

Step 10

Clear the screen by entering the following command:

clear

You can also scan for a range of ports. To do this, you need to specify the first and the last port. Type the following command:

Press Enter.

Notice the output. Only port 53 is open.

Task 3 - Active vs. Passive Enumeration

Enumeration can be done using the active and passive fingerprinting of a system. During active Enumeration, there is an interaction with the target. You can use a tool such as Nmap to perform active Enumeration.

When you execute a command to determine the operating system of a remote host, packets are sent to the remote host, and the response is received, which is compared with the list of operating systems contained in Nmap. After the response is received, then Nmap provides the closest match.

To perform active and passive Enumeration, perform the following steps:

Step 1

Ensure you have powered on all the devices listed in the introduction and connect to PLABKALI01. Click the Terminal icon in the left pane if the terminal is already not opened.

To enumerate a remote system, type the following command:

nmap -O 192.168.0.3

The -O parameter provides more options for operating system detection

Step 2

The output of the command is displayed. In this output, the operating system version has been detected. Nmap also detects the open ports and also the kernel version.

Step 3

clear

The output of the command is displayed.

Clear the screen by entering the following command:

Finding Analysis and Weaponization

Weaponization is a method of using the information gathered in the reconnaissance phase and then use it as pointers for proceeding further. With the help of weaponization, you can decide what actions to be performed next. Weaponization helps to filter useful information from the gathered information carefully.

The information must be filtered carefully based on some of the key criteria:

  • Scope of what needs to be tested
  • Targets involved in the testing
  • Capabilities of the team who will be testing

When filtering and analyzing information, you need to keep the big picture in mind. The information can be filtered into three categories:

  • Information that can be used immediately
  • Information that is not useful at present but can be used later
  • Information that has no use and can be discarded

Filtering and analyzing information becomes a critical task. If not done with caution, the testing may cause delay or direct it in the wrong directions.

Content of Interest

In passive reconnaissance, you are likely to find quite a bit of information. Some of such information is likely to be:

  • IP addresses and sub-domains
  • Organization’s partner’s or vendor’s Websites
  • Employee information
  • Social Engineering
  • Systems and devices

Let’s look at each one of them in detail.

IP Address and sub-domains

In the passive reconnaissance, you are likely to discover the public IP addresses that are used by various services, such as Web servers, FTP servers, and messaging servers. This information can be useful as it might serve as an entry point to a private network. This is likely to happen when a service hosted on the public IP address is vulnerable and can be exploited to gain entry into the private network.

You can also enumerate DNS to figure out the subdomains if they exist. A subdomain is likely to represent an internal network, such as intranet.plab.com. This might be a good option to explore and can be used in testing. If any vulnerabilities may exist, you can exploit them to gain entry into the private network.

Organization’s partner’s or vendor’s Websites

An organization’s partner or vendor’s Website may or may not be useful in testing. If the vendor is currently not active with the organization, then there is no point in including its Website in the testing. However, if the vendor or partner is currently working with the organization, then it will depend on the scope of testing.

A vendor or partner’s Website, however, may reveal information about the organization, its key personnel, and their E-mail IDs, which can be mentioned in reference articles, for example. Such information can be useful in testing.

Employee Information

Information about an employee can be gathered using different mediums. For example, you can explore some of the mediums, such as:

  • Social media Websites, such as Facebook, Twitter, LinkedIn and so on
  • PGP E-mail searches using various tools, such as theHarvester
  • Organization’s Website

There are enough methods that can be used to gather information about employees. Some of the key information that you can target is:

  • E-mail addresses
  • Phone numbers
  • Designations
  • Nature of their job profile - technical or non-technical
  • Office location
  • Date of birth
  • Interests
  • Relationships

Social Engineering

The gathered information about individuals can be used in social engineering. For example, you can attempt a spear-phishing attack and design the phishing E-mail in a manner that makes the victim believe what has been written. This can happen because you have used some of the personal points, such as bank account information, in the E-mail.

Systems and devices

Using various tools, such as Maltego, you can discover the technology being used in the organization’s public Website. Most simply, you can discover the type of Web servers being used. Once you discover servers such as Internet Information Services (IIS) being used, you can focus on targeting the IIS server in active reconnaissance.

You can also research and discover the vulnerabilities that may exist. In some cases, you will find that the organization has not patched their servers. This gives you a good chance to exploit open vulnerability during testing.

Preparing for Next Steps and its Guidelines

After reconnaissance, you need to prepare for the next steps. Here are some of the key guidelines that you can use:

  • Weaponize the information carefully and filter the information - something that you can use right away, something can be kept aside for future, and something that needs to be discarded.
  • Information that is not required should be discarded immediately. The focus should be on the information that helps you achieve actionable tasks.
  • Filter the information within the scope of testing. Do not filter based on individual tasks as the information should not be used in isolation.
  • Filter out information on the public and private IP addresses. Public IP addresses may belong to Web servers, which can be targeted later.
  • Find and filter the information about the individuals who work for the organization. This information can become handy in conducting social engineering attacks.
  • Find and filter the systems, servers, devices, routers information. These will become the potential targets in testing, with the exception that the scope permits them to be used as targets.
  • Carefully use the gathered information within the scope of testing.
  • Appraise the entire information with the gathered information and its analysis.

Document the findings for later use. A document with the complete findings can be helpful for the team at a later stage or in another project.
